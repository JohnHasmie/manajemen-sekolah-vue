Trade-off setiap opsi

  • Opsi A · Stacked KI rows paling kompak
    Card tetap satu baris, dua angka di kanan disusun vertikal (KI 3 di atas, KI 4 di bawah). Pro: tinggi card hampir sama dengan UI lama (~64dp). Con: angka jadi lebih kecil (15pt vs 22pt di detail); kalau partial-fail, perlu warna merah pada angka KI 4.
  • Opsi B · KI strip di bawah
    Top row letter+nama+chevron, lalu di bawahnya dua cell sejajar (mirip card detail TA.29 tapi padding lebih ketat). Pro: konsisten 100% dengan card detail page. Con: card ~30% lebih tinggi → list jadi panjang kalau mapel banyak.
  • Opsi C · Two badges horizontal rekomendasi
    One-row card; di kanan ada dua pill mini (KI 3 cobalt + KI 4 violet), masing-masing label tag di atas + angka di bawah. Pro: tetap kompak, dua angka readable, warna brand jelas, mudah skim 10 mapel. Con: butuh ~95px lebar untuk dua badge → mungkin perlu trim guru/predikat line jadi 1 line saja (sudah).

Konvensi visual (sama untuk semua opsi)

  • KI 3 = cobalt #1B6FB8, KI 4 = violet #7C3AED.
  • Cell failing (skor < KKM) override jadi merah (border + angka).
  • Letter badge tetap di kiri, ukuran 38–40dp, palette tone (A=green, B=blue, C=amber, D=red) — di-derive dari rata-rata KI 3 & KI 4.
  • Card tetap tappable → buka showParentRaporDeskripsiSheet() yang sudah kita bangun di TA.30 (deskripsi capaian per mapel).
  • Hero gradient parent (brandDarkBlue → brandAzure), KPI strip (Rata-rata / Kehadiran / Mapel) tidak berubah.

Backend & database

  • 0 migration — semua data (knowledge_score, knowledge_predicate, skill_score, skill_predicate, kkm) sudah ada di payload /parent/raports. UI lama cuma tidak menampilkannya.
  • PDF tidak terdampak — output cetak dari list maupun detail tetap pakai template yang sama yang sudah dual-column sejak TA.25.

Pilihan saya

  • Rekomendasi: Opsi C — paling seimbang antara densitas info, kompaksitas (~64dp tinggi card), dan konsistensi brand. Skim 10 mapel masih cepat.
  • Alternatif: Opsi A kalau prefer angka tersusun vertikal & label tersembunyi di tag pill (paling minimal).
  • Hindari: Opsi B kecuali jumlah mapel umumnya < 6 — kalau 10 mapel ditampilkan dengan layout B, halaman akan jadi 1.5× lebih panjang dari Opsi C dan user perlu scroll lebih banyak sebelum sampai ke CTA "Lihat rapor lengkap".
