Isi Raport · Nilai Tab — Pengetahuan (KI 3) + Keterampilan (KI 4)

The Nilai tab in the wali kelas Isi Raport flow today exposes only ONE score per mata pelajaran, even though the paper raport (and the backend schema) require two separate scores: Pengetahuan (KI 3) and Keterampilan (KI 4). The backend's raport_subjects table already has knowledge_score, knowledge_predicate, knowledge_description, skill_score, skill_predicate, skill_description columns and the controller already persists all six — only the frontend renders/edits a single side. This redesign extends the mapel row card to show both scores side-by-side and rewires the edit sheet into two stacked sections (one per KI). No backend or database work required.

Dual-section edit sheet. Same gradient head + drag handle pattern as every other sheet. Inside the body, two `.nilai-section` cards stack vertically: cobalt-tinted icon frame for Pengetahuan (KI 3), violet for Keterampilan (KI 4). Each section has: (1) Nilai numeric input with an "≥ KKM" / "< KKM" auto-badge that turns green/amber based on score, (2) Predikat chip-row auto-selected from score (admin can override by tapping), (3) Deskripsi textarea pre-filled with the per-predikat template the wali kelas can edit. Footer Batal/Simpan via shared BottomSheetFooter.

The fix is entirely frontend: render the second score, plumb the second set of callbacks, and let the existing save mutation carry the new fields.

Auto-predikat helper (shared)

String predikatFromScore(num? score) {
  if (score == null) return '';
  if (score >= 90) return 'A';
  if (score >= 75) return 'B';
  if (score >= 60) return 'C';
  return 'D';
}

Auto-fill rules: when the teacher edits the score field, the predikat chip flips to the matching letter. If they tap a different predikat afterwards, the explicit choice wins and a small "Manual" hint appears under the chip-row so they know it's no longer auto. Same logic for both Pengetahuan and Keterampilan independently.
