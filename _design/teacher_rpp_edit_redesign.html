Visual confirmation — sheet auto-pops on save. Section card briefly tints cobalt to highlight what changed. Snackbar confirms which section was saved. Other sections untouched (because PATCH only sent that one key).
Why this works better than the legacy "edit mode"
  • Smaller blast radius. One section save = one PATCH key. Backend's UpdateLessonPlanAction already merges format_data partials, so editing Tujuan can't accidentally clear Penilaian.
  • Regen sits next to the edit it modifies. The violet Generate ulang button is a footer action on the same sheet that shows the section. No separate regen sheet for the common case (the multi-section regen sheet J from the original mockup is still there for the rare bulk regen).
  • Identity edit reuses setup-sheet patterns. Same chip-style fields the teacher saw when creating the RPP. Drops LessonPlanFormDialog for the structured formats.
  • No "edit mode" state. The detail page has one shape always. Eliminates a whole class of "what mode am I in?" bugs and makes back-button behaviour obvious (always returns to the list).
  • Format-aware out of the box. The pencil affordance only renders for sections in format_section_keys. K13 → 5 pencils. RPP 1 Halaman → 3. Modul Ajar → 6. File → no pencils (file detail uses upload sheet for replacement, header pencil disabled).
  • Implementation impact — two new sheet files (lesson_plan_section_editor_sheet.dart, lesson_plan_identity_edit_sheet.dart). Drop the global isEditing toggle from AiRppDetailScreen. Drop AiRppEditorView entirely (replaced by per-section sheet). Keep LessonPlanFormDialog only for the file-format ganti-file flow until LessonPlanUploadSheet grows an edit mode.
