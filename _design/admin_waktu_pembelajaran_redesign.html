Apa yang berubah

  • Frame A · Hub — Tambah KPI strip overlap (Hari aktif / Total jam / Durasi rata-rata) di antara header dan list, mirip layar Pengaturan Umum & Mata Pelajaran detail. Hapus section header lama Jam Aktif Harian yang ber-dua-line padat → ganti dengan section header pattern (32dp icon + dual-line text) yang dipakai di seluruh Sistem.
  • Day cards sekarang lebih dense (44dp tinggi) dan memuat status pill (AKTIF / KOSONG) + range waktu (07:00 – 13:45) di samping jumlah sesi.
  • Frame B · Day sheet — hero gradient pakai admin navy, bukan azure parent. Format waktu tanpa detik (07:00, bukan 07:00:00). Tambah duration chip (45 mnt) per row supaya admin cepat memverifikasi panjang sesi. Tombol Tambah dipindah ke footer pakai BottomSheetFooter pattern + tombol baru Salin untuk bulk copy.
  • Frame C · Edit Sesi sheet — gradient admin navy (bukan biru lama). Field pakai BrandTextFormField pattern: icon box di kiri, label uppercase, value bold. Bagian Mulai & Selesai disusun side-by-side. Tambah Durasi auto-pill (45 menit) yang otomatis dihitung dari rentang waktu, dan warning hint kalau bentrok dengan sesi lain.
  • Frame D · Empty state — hari yang belum punya sesi sekarang punya dual CTA: Salin Hari + Tambah Jam, mirip pattern empty state di TR.H (Jadwal).
  • Frame E · Salin Dari Hari Lain — sheet baru. Pilih hari sumber dari list yang sudah aktif, tap untuk seleksi (single-select), tampilkan jumlah sesi + range. Warning kuning kalau target sudah punya sesi (akan diganti).
  • Frame F · Hapus konfirmasi — pakai ConfirmationDialog shared, dengan context card di tengah (icon clock + label "Jam ke-3 · Senin · 09:00 – 09:45") supaya admin yakin sesi mana yang dihapus.

Shared component yang dipakai

  • BrandPageHeader(role: 'admin') — navy gradient untuk hub.
  • BrandPageLayout — overlap KPI 22dp.
  • BrandKpiStrip — Hari aktif · Total jam · Durasi (3 cell, pill di bawah).
  • DashboardListTile — day row dengan colored 42dp icon + dual-line text + trailing status pill + chevron.
  • AppBottomSheet.show() — day sheet (B), edit sheet (C), empty sheet (D), salin sheet (E). Semua pakai header navy.
  • BottomSheetFooter — Batal/Simpan, Tambah/Salin, dst. Tidak lagi hand-rolled.
  • BrandTextFormField — Jam ke-, Mulai, Selesai. Pakai icon-box pattern.
  • ModernDatePicker (atau platform showTimePicker) — buat picker Mulai/Selesai.
  • ConfirmationDialog — hapus sesi.
  • EmptyState — hari kosong di dalam day sheet.
  • FilterChipGrid (opsional) — bisa dipakai di Frame E kalau mau multi-select hari sekaligus.

Backend & database

  • 0 migration wajib. Endpoint sudah ada: GET /settings/lesson-hours, POST /settings/lesson-hours, PUT /settings/lesson-hours/{id}, DELETE /settings/lesson-hours/{id}.
  • Optional backend additions (kalau Salin Dari Hari Lain mau efisien):
    • POST /settings/lesson-hours/copy body { from_day_id, to_day_id, replace_existing: bool } — bulk insert dalam 1 round-trip. Tanpa endpoint ini frontend bisa loop CREATE per sesi (acceptable untuk < 12 sesi).
    • POST /settings/lesson-hours/bulk-delete body { day_id } untuk wipe-and-replace. Optional.
  • Validasi overlap sudah ada di backend (sesi tidak boleh tumpang tindih), cukup surface error 422 sebagai SnackBar.
