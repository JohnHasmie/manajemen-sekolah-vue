Keuangan · Admin · Tagihan list + Laporan per kelas + Matrix per kelas

Three surfaces re-laid-out around the same problem: too many duplicated rows because each payment-type × class × academic-year combination becomes its own row. The new flow groups by academic year first (default to current AY), then by Tingkat → Kelas. Matrix per kelas drops the empty grid in favour of payment-type tabs + status pills, plus a card-view toggle for narrow screens.

Sistem
A1 — Tagihan grouped by Tingkat → Kelas. Default AY = current (Aktif chip), collapsing the noisy duplicate rows we saw before (SPP 8A 2024/2025, SPP 8A (2024) 2025/2026, SPP 8A 2025/2026 all in one list). Each Tingkat group expands to show its classes; tapping a class drills into the same C1/C2 matrix. KPI strip overlaps the header per BrandPageLayout.kpiOverlapHeight. The "Aktif" chip dot mirrors the Pengumuman event live indicator.
A2 — Tagihan grouped by Jenis. Alternative grouping when admin cares about "how is SPP collection going across all classes" — bill-group card shows the jenis pill, span ("5 kelas aktif"), progress bar, and outstanding count. The two groupings (A1 vs A2) can co-exist behind a single chip-row toggle Per kelas · Per jenis in the filter row.
B1 — Laporan per kelas with Tingkat sections. Replaces the flat list "6A (2024), 6B (2024), 7A, 7A (2024)" pattern by surfacing the AY scope as a header chip first. Each class row carries the wali kelas + outstanding pill so admin doesn't need to drill in to see severity.
B2 — "Semua tahun" mode. When admin flips the AY chip to "Semua tahun", AY groups collapse by default; the active year stays expanded so primary workflow isn't slowed down. Archived years show a single progress pill.
C1 — Matrix view. Sticky student name column, monthly status pills with iconography (✓ lunas, ✕ belum, ! tempo, ⏳ pending, – belum berlaku). Drops the all-empty 12-month grid by paginating per payment type (SPP→5 bln visible) — bigger cells, better readability. Tapping any pill opens the existing detail sheet redesigned earlier.
C2 — Per-siswa card view. Toggleable from the same screen. Each card shows the student avatar, NIS, status summary (e.g. 2/4 lunas), and a 6-cell strip representing the current jenis's periods. Friendlier on small phones than horizontal-scrolling a 12-column matrix.
Behavioural rules across all frames. (1) AY chip is sticky-by-default to current AY — admin doesn't need to filter every session. (2) Jenis tabs persist when toggling between Matrix and Per-siswa views. (3) Search bar always scoped to the visible jenis tab. (4) Single source of truth for cell-tap: routes to the existing detail bottom sheet (Lunas → detail, Belum → options sheet with Bayar Manual + Lihat Detail). (5) KPI counts always reflect the active AY × Jenis combination so the numbers always tell the truth about what's currently on screen.
