Tahun Ajaran & Pengaturan Umum — Admin Redesign

Five frames + backend changes. The current "Tahun Ajaran" entry on the Sistem hub routes to a screen titled "Pengaturan Umum" that only renders Informasi Sekolah, with no actual academic-year management surface and an old hand-rolled AlertDialog for editing. This redesign (a) renames the hub correctly, (b) adds a dedicated Kelola Tahun Ajaran screen with current-year hero + arsip list + FAB, (c) replaces the Edit dialog with brand AppBottomSheet, (d) adds a Tambah/Edit Tahun Ajaran sheet wired to the existing backend, and (e) adds a high-stakes ConfirmationDialog for the "Aktifkan" cascade. Every dimension below matches the audited Flutter brand components verbatim.

Backend changes

Most of the redesign reuses existing endpoints. Three gaps need backend work to fully wire the mockup: (1) a semester field per academic year so the hub can show "Ganjil/Genap" without parsing the year string, (2) a status enum that supports an explicit archived value to match the UI's three-state pill, and (3) a single bulk KPI endpoint so Frame B's KPI strip + per-row counts come from one round-trip instead of N.

1. Migration — extend academic_years schema ALTER

File: database/migrations/2026_05_22_000001_extend_academic_years_with_semester.php. Already exists: id, school_id, year, current, status, start_date, end_date, timestamps. 

Schema::table('academic_years', function (Blueprint $table) {
    // Embedded semester so the UI does not have to parse the year string.
    // Values: 'ganjil' | 'genap'. Nullable for legacy rows.
    $table->string('semester', 10)->nullable()->after('year');

    // Promote status to a 3-state enum: active | inactive | archived
    // (today only active/inactive are used). Existing 'inactive' rows
    // remain 'inactive'; the controller's archive endpoint flips to
    // 'archived' explicitly.
    $table->index(['school_id', 'status'], 'ay_school_status_idx');
});

2. Model — AcademicYear.php ALTER

protected $fillable = [
    'school_id', 'year', 'semester',  // ← new
    'current', 'status', 'start_date', 'end_date',
];

public function isArchived(): bool
{
    return $this->status === 'archived';
}

public function scopeActive($q)   { return $q->where('status', 'active'); }
public function scopeArchived($q) { return $q->where('status', 'archived'); }

3. Controller — new + extended endpoints NEW EXTEND

MethodRouteStatusNotes
GET/academic-yearsEXTEND Already returns rows + counts. Add semester field to response payload. Sort by start_date DESC, year DESC.
GET/academic-years/kpi-summaryNEW One-shot summary for Frame B. Returns { total, current_count, archived_count } so the KPI strip doesn't need to count client-side.
POST/academic-yearsEXTEND Accept semester in payload. Validate enum: ganjil|genap. Auto-default status to inactive.
PUT/academic-years/{id}EXTEND Accept semester. Reject if year is archived (return 422 "Tahun ajaran arsip tidak bisa diubah").
POST/academic-years/{id}/archiveNEW Sets status = archived. Reject if current = true (must transfer current first). Triggers notification job to guru & wali.
POST/academic-years/{id}/unarchiveNEW Sets status = inactive. For the "Pulihkan" overflow action.
PUT/academic-years/{id}/set-currentEXTEND Already exists. Extend to auto-archive the previously-current year and dispatch the cascade notification job that Frame E warns about.
DELETE/academic-years/{id}EXTEND Reject if any FK rows reference this academic_year_id (classes, student_classes, bills, etc.). Return 422 with a friendly message — admin must archive instead.
