Schema & endpoint diff

1 · Migration baru

Tambah ke tabel announcements:

  • + event_at timestamp NULL — kalau set, pengumuman berubah jadi acara.
  • + event_end_at timestamp NULL — durasi opsional (untuk all‑day & rentang).
  • + event_has_time boolean DEFAULT true — jam=opsional, jadi kalau false UI tampilkan "sepanjang hari".
  • + event_location varchar(160) NULL — text bebas (tidak terikat ruangan).
  • + reminder_offsets jsonb NULL — array menit sebelum (mis. [1440, 60, 0] = 1 hari, 1 jam, saat mulai).
  • ~ pinned_until DEFAULT (event_at) via update path — sematan auto-lepas hari kejadian.

Tabel pendamping announcement_reminders (queue audit):

  • + id uuid
  • + announcement_id uuid → announcements.id
  • + offset_minutes int
  • + fire_at timestamp — index pada (fire_at, status)
  • + status enum('pending','sent','failed','skipped')
  • + recipients_total int
  • + recipients_delivered int

2 · Endpoint

POST /announcements terima event_at, event_has_time, event_location, reminder_offsets[]
PATCH /announcements/{id} ubah Acara setelah publish — re-queue reminder otomatis
GET /announcements?has_event=1&from=YYYY-MM-DD&to=YYYY-MM-DD untuk kalender admin
GET /announcements/upcoming-events 3 acara terdekat — dipakai dashboard hero strip semua role
POST /announcements/{id}/personal-reminder user (wali/guru) tambah reminder lokal · offset_minutes

3 · Worker / scheduler

  • Job DispatchAnnouncementRemindersJob jalan tiap 1 menit, pick semua announcement_reminders dengan fire_at ≤ now() & status='pending'.
  • Per row → render template push + insert ke notifications tabel untuk audiens yang sudah diresolusi waktu publish.
  • Setelah dispatch → status = sent; failure → failed dengan retry max 3 dari worker yang sama.
  • Indeks gabungan (fire_at, status) bikin scan tetap O(reminders due) walau tabel 100k+.

4 · Cron snapshot harian

  • 00:05 — auto-lepas is_pinned untuk pengumuman yang event_at < today.
  • 00:10 — kirim "daily digest" ke wali dgn ≥1 acara hari ini (opsional di Phase 2).

Bug fix paralel — Waktu Pembelajaran (Tambah Sesi)

Jam tidak berubah setelah dipilih · root cause

Di session_time_picker_mixin.dart:18–24, callback setModalState(() { startTime = picked; }) hanya mengubah parameter fungsi lokal buildTimeFields. Variabel asli ada di scope _buildAddEditSessionSheet dan tidak ikut termutasi. Jadi setiap kali StatefulBuilder rebuild, mixin dipanggil ulang dengan nilai lama dari scope luar, dan UI tampak tidak berubah.

Perbaikan: hoist startTime / endTime ke dalam scope yang sama dengan setModalState (di dalam _buildAddEditSessionSheet, sebelum return StatefulBuilder(...)), dan pindahkan closure pickTime ke dalam builder: sehingga mutasi setelah setModalState dibaca lagi di build berikutnya. Tidak perlu ubah skema; murni perbaikan state-management Flutter.

Catatan implementasi

  • Backward compatible — pengumuman tanpa event_at tetap render seperti sekarang. Tidak ada perubahan yang memutus existing data.
  • Per-role reach — reminder dispatcher pakai audience_matrix existing untuk resolve daftar penerima sekali (saat publish/ubah), simpan jumlah ke recipients_total agar audit cepat.
  • Add-to-calendar — di guru & wali, button generate file .ics via file_picker/share_plus — tidak perlu integrasi kalender native.
  • Phase 1 vs Phase 2 — Phase 1: schema + compose + dispatcher + list/detail di 3 role. Phase 2: kalender admin penuh, RSVP hadir (guru), digest harian (wali), reminder personal kustom.
