Admin · Jadwal Mengajar — Interactive redesign

Today the admin Jadwal page is a flat list of cards with a hidden matrix toggle — no overview of when the school is busy, no visual sense of clashes, no quick way to shift one class's session to a different slot. This mockup proposes a 3-view interactive scaffold (Grid · List · Matrix) on a brand-consistent navy header, with drag-to-reschedule, click-to-edit sheet, bulk-select mode, and conflict detection baked into the calendar grid itself.

Today (live)

Single flat list of cards (one row per session) — no calendar overview.
Matrix toggle exists but matrix is dense, read-only, hard to read on phone.
No visual indication of conflicts. Admin must mentally scan for them.
Add/edit dialog is a full-screen form with no conflict pre-check.
No bulk operations — can't reschedule a week's worth of sessions when a teacher swaps shifts.
Drag-to-reschedule is not possible; admin must delete + recreate.
Filter sheet exists and works — keeping it (Frame G).
Excel import/export exists — keeping it, just surfacing better (Frame H).

After redesign

3-view toggle (Grid · List · Matrix) — Grid is new and shared with most calendar apps.
KPI strip shows Total Sesi · Hari Ini · Bentrok count at a glance.
Conflict blocks render in red right on the grid; rows in List view get red "Bentrok" pills.
Add flow runs live conflict pre-check; disables Simpan when slot is busy.
Long-press → bulk-select; reschedule / change teacher / delete N rows at once.
Drag-and-drop reschedule with ghost slot + Undo toast.
Tap-to-edit quick sheet for fast inline changes (no full page hop).
Backend impact: needs a /schedules/conflicts endpoint that takes (day, start, end, exclude_id) and returns clashes. Other endpoints stay identical.
