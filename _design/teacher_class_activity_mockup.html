Notes for review

  • One detail surface, three buckets. Frame A is the read-write detail (today / current academic year). Frame E is the same scaffold in lock mode (past TP). Frame B is the FAB-driven add sheet, Frame C is the edit sheet (pre-filled, with class+mapel locked).
  • 3-cell KPI overlap. Top-of-detail card breaks per-activity progress into Siswa · Submit · Belum. Numbers come from the Activity ↔ Submission join, fetched alongside the activity itself.
  • Type tiles, not dropdown. The add/edit sheets show four big tiles (Tugas, Aktivitas, Ujian, Catatan) with icon + label + 1-line desc. Selected tile fills with brand-cobalt; tap toggles. Faster than a dropdown for a 4-item set.
  • Sticky save footer. Bottom of every sheet — Batal (white outline) + Simpan (cobalt). Edit/Add use the same shape; Detail flips to Hapus + Edit so the destructive action is left-of-primary, never primary itself.
  • Quick actions sheet (Frame D). Salin tautan / Duplikat / Ekspor PDF / Hapus. The duplicate row uses violet because it spawns N new activities — distinct from the green export and red delete.
  • Locked variant. Past academic year flips kicker to ARSIP, dot to slate, replaces the ⋯ icon with ⤓ download. Edit/Hapus footer is hidden; only the back arrow + export remain.
