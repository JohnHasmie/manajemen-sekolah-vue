Catatan implementasi
  • Format adalah sumbu utama. Frame B (Format Chooser) muncul saat tap FAB + — guru memilih K13 / RPP 1 Halaman / Modul Ajar / Upload File terlebih dahulu. AI vs Manual menjadi sub-axis (toggle) di Frame C, bukan dialog terpisah.
  • Satu chrome, empat skin. Header & KPI overlap pakai BrandPageLayout + BrandPageHeader yang sama; perbedaan hanya warna gradient (indigo / emerald / violet / slate), kicker, dan format-badge pada list card.
  • Section schema per format. Editor (Frame I) dan regen sheet (Frame J) membaca section keys dari format yang dipilih. K13 punya 5 section, RPP 1 Halaman punya 3 (compressed), Modul Ajar punya 6 (Kurikulum Merdeka), Upload File hanya metadata + file.
  • Upload File flow. Frame G memakai upload-drop + metadata wajib (kelas, mapel, bab); detail (Frame H) menampilkan file-card non-editable + tombol Ganti file. Bukan format isi — tapi tetap entri di list yang sama.
  • Konversi file → RPP digital. Detail upload (Frame H) menampilkan ai-card opsional: AI bisa baca PDF dan susun ulang ke K13 / 1 Halaman / Modul Ajar (defer ke Phase 2 backend).
  • Filter sheet (Frame L) dapat 2 section baru: Format (multi-select 4 chip) dan Metode (AI / Manual / Semua). Format chip menggunakan dot warna sesuai brand format-nya.
  • List card (Frame A) menampilkan fmt-badge kecil di scope-row paling kiri — itulah yang membuat 4 format dapat dibedakan dalam 1 detik.
  • Backend yang bisa ditunda (per arahan): kolom format di tabel lesson_plans (enum: k13, rpp_1_halaman, modul_ajar, file), kolom file_path + file_size + file_mime untuk upload, dan format-aware system prompt di LessonPlanService.
