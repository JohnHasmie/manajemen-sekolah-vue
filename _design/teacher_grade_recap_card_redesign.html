Rekap Nilai · Card Redesign

The current overview cards stack subject name, class pill, an empty progress bar, and a row of red "0 nilai · siswa · Perlu input" pills — three red signals when really nothing is wrong, just empty. The redesign separates identity, stats, and progress into clear bands; uses muted colors for empty states (no false alarms); and keeps the rata-rata badge consistent across rows.

What's changing & why
  • Stats grid replaces the pill row. 3 cells (Bab · Nilai · Rata-rata) on a soft slate-50 chip with hairline dividers. Equal width, balanced visual weight, no reds-by-default.
  • Status tag in the header. One small all-caps pill (Belum mulai / Sebagian / Hampir tuntas / Tuntas) replaces the "Perlu input" red alarm. Empty state is slate-tinted, not red — accurate signal.
  • Empty state has a CTA. Instead of a 0% red bar + red badge, we show a striped muted bar and a "Mulai input ›" affordance — feels like an open task rather than a broken card.
  • Avg lives inside the stats grid. Drops the floating right-side avg badge so all rows have identical horizontal alignment, no skipped column on rows without data.
  • Class pill simplified. Slate-tinted instead of red — semantic only when something's actually wrong.
  • Progress bar paired with %. Percent text right-aligned to the bar, color matches: green ≥80, amber 40-79, red <40 (only when there IS data — otherwise muted).
  • Wali-kelas line. Subject teacher name shown with a person icon under the head, separated from the student count with a thin · dot. Same line, no extra row.
