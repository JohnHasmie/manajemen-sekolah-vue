Rekap Nilai · Mockup v1

Seven frames covering the teacher Rekap Nilai flow. Replaces the legacy TeacherPageHeader overview + dialog-style recap matrix with a brand-migrated stack: BrandPageLayout overview (matrix & list views), full-screen recap detail with frozen-name column + chapter-grouped columns, sticky save bar that surfaces only when the matrix is dirty, add/rename chapter sheets, an empty state for fresh classes, and an archive variant for past academic years. Score input flows from Catat Submit into this same matrix automatically (Phase 3).

Implementation notes
  • Frames A & B share the same overview screen — view toggle in header switches between matrix grid and dense list. View preference persists in LocalCacheService per the existing rekap_nilai_view_mode key.
  • Frame C replaces the legacy modal/dialog with a full-screen route. Matrix uses FrozenColumnTable (already in core/widgets) for the sticky Siswa column; chapter heads tint blue/violet alternating. Score cells: ≥KKM green, edge (=KKM) amber, <KKM red, blank dash for missing.
  • Save bar is a sticky bottomNavigationBar that only mounts when the matrix is dirty. Mirror Presensi's "5 perubahan belum disimpan" pattern. Batal reverts via GradeRecapUnsavedChangesDialog.
  • Frames D & E reuse AppBottomSheet + BottomSheetFooter. Add Chapter wires to add_chapter_sheet.dart (already exists, needs brand polish). Edit Chapter wires to grade_recap_chapter_rename_dialog.dart — promote it from dialog to sheet for consistency.
  • Frame F uses EmptyState with a custom CTA; the info card below references the Phase 3 sync from Catat Submit so teachers know scores can flow in automatically.
  • Frame G is the read-only variant — no save bar, slate dot, kicker reads ARSIP, top-right icon flips to download for PDF export. Matches the Kegiatan Kelas Frame E archive pattern.
  • Subject letter avatars use a palette rotation by subject family (Math→blue, Sci→violet, Lang→green/amber, Art→rose, Civic→indigo). Same rotation as the Buku Nilai admin overview.
  • Score input → Buku Nilai sync from class_activity Phase 3 already lands in the same grades table this matrix reads from. Edits done here propagate back to Catat Submit views via the same assessment row.
