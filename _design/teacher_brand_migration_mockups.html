Wali-kelas-only screen. Mengajar role hides this from the bottom nav (or shows an empty state with a switch-to-wali CTA).

Backend changes I'd ship alongside

  • /api/teacher/dashboard/kpis — single endpoint returning the per-screen KPI bundles for the active teacher (sessions today, RPP status counts, assessment counts, etc.) so each screen doesn't need its own GET.
  • Per-screen endpoints get a ?include=kpi param as a fallback (the screen can stand alone if the bundle endpoint fails).
  • Homeroom assignments → list, not single. Where the API currently returns one homeroom class for a teacher, expand to homeroom_classes: [{id, name, student_count}] so the multi-wali-kelas RoleToggleChipRow has its data.
  • Cache: KPI bundle is cacheable for 60 s with the existing realtime poll loop, same TTL as the parent dashboard.

Notes for review

  • FAB is removed on Rekap Nilai overview, Jadwal (read-only for teachers), and Raport overview. The other 6 keep their FAB for "tambah".
  • Search bar is gone. Pre-migration teacher screens had an inline search; the brand pattern keeps search inside the Filter sheet (open via the gear icon). If you want the inline bar back I can put it in the body above the section header.
  • Realtime dot sits inline next to the title (6×6 px, green). The "last sync 2 minutes ago" text was retired in the compact v2 pass per the parent-role redesign.
  • Role chips ellipsize when ≥4 chips fit in the strip; on overflow the strip becomes horizontally scrollable. Single-role teachers (no homeroom) get no chip strip at all.
  • Filter chips always open the same shared filter sheet — tapping any chip is equivalent to tapping the gear icon. The chip body just previews the active value.
