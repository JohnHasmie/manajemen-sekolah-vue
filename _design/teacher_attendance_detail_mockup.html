Same brand language as the parent surfaces — gradient header with realtime dot, KPI overlay, sticky save footer. The student row is the core primitive: a single inline picker on the right-hand side flips between status states without ever opening a popup.

Notes for review

  • One screen, two modes. Frame A is the "new" flow (FAB tap on Presensi); Frame B is "edit" (existing card tap). Same scaffold, different starting state. Title flips between Ambil Presensi and Edit Presensi based on whether `attendance_id` is present.
  • Big-target 4-segment picker (Frame A v2). The student card is two stacked rows: avatar + name on top, then a full-width 4-segment picker (Hadir / Sakit / Izin / Alpa) underneath. Each segment is 46dp tall and ~78dp wide — well above iOS 44pt and Material 48dp minimum hit targets, so a teacher can mark students with their thumb without zooming. Active segment fills with its colour and turns its glyph dot solid; inactive segments stay outlined. Saves a UI roundtrip for the dominant case (mark a few absent, save).
  • Pill + sheet (Frame B). Once a session is saved, the right-hand affordance becomes a single colored pill ("Hadir", "Sakit", etc). Tapping the pill opens Frame E — a per-student sheet with the 4 status tiles + a notes field. This keeps the list tighter when reviewing.
  • KPI strip is live. The 4-cell counts at the top update as the teacher taps statuses. Same overlap pattern as the Presensi list KPI (`BrandPageLayout.kpiOverlapHeight`).
  • Quick actions (Frame C). Mark all Hadir, fill remaining as Alpa, copy from last session, reset, change date/slot. Keeps the bulk-mark chips on the body simple and shoves power-features into the sheet.
  • Date & slot picker (Frame D). Tap ✎ on the context strip to open. Calendar shows green dots on dates that already have records; below is a list of today's scheduled sessions. Picking a green date opens edit mode for that session; picking a future-or-empty date pre-fills today's mengajar slot.
  • Sticky save footer. Always shows "X / N dinilai" so the teacher knows when the form is complete. Save button is green when there are unsaved changes, slate when there aren't (Frame B). On save, the sheet pops back to Presensi and the card flips from "Belum" amber to a percentage green.
  • Read-only fallback (Frame F). Past academic years lock editing. Header dot is grey, banner explains why, no save footer, only an export icon (⤓) in the action slot.

Backend changes I'd ship alongside

  • GET /api/attendance/session-detail?class_id&subject_id&lesson_hour_id&date — returns a session metadata block + the student list with their current status + notes for that session, in one call. Replaces the current "load class students, then load attendance, then merge" three-call flow.
  • POST /api/attendance/session-detail/save — atomic upsert of all student statuses for the session. The body is a list of {student_id, status, note}. Backend deletes-then-inserts inside a transaction; mobile sends only the rows that changed since load.
  • GET /api/attendance/calendar?month=2025-04&teacher_id=… — returns an array of {date, has_records} for the month so the date picker (Frame D) can paint green dots without N round-trips.
  • has_records rolls into the existing teacherSummary so the Presensi card's "Belum / 100%" pill reflects this session's saved status without a refetch.
