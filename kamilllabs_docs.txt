Page 41. Overview & Architecture
1.1 System Overview
KamillLabs Edu AI is a comprehensive AI-powered educational platform built on Laravel 12 that 
provides three core features for Indonesian schools: Learning Recommendations, Teaching 
Material Generation, and RPP (Lesson Plan) Generation.
The system supports multiple AI providers (Anthropic Claude and Google Gemini) with dual-
mode operation supporting both synchronous and asynchronous processing via Laravel 
queues. Providers can be switched via environment configuration without any code changes.
1.2 Architecture Patterns
The codebase follows a clean, layered architecture with single-responsibility layers:
Layer Location Responsibility
Controllers Http/Controllers/Api/Thin orchestrators that delegate to other 
layers
Form Requests Http/Requests/{Feature}/ Input validation and rate limit pre-checks
Actions Actions/{Feature}/ Business logic for writes and mutations
Repositories Repositories/ Database read abstractions
API Resources Http/Resources/ Response formatting with success wrapper
Services Services/AI/ AI integration and generation logic
Models Models/ Eloquent models with UUID primary keys
Enums Enums/ PHP 8.4 string-backed enums

1.6 Rate Limiting

places: Form Request validation (early rejection) and Service layer (safety net for queue jobs).

Provider Registration
The AIServiceProvider uses a match expression to resolve the correct implementation based on  
the configured provider:
Provider ENV Value Service Class API Key Transport
Anthropic Claude anthropic (default) ClaudeAIService Header: x-api-key
Google Gemini gemini GeminiAIService Query param: ?key=
Shared Interface
Both providers implement the same AIServiceInterface contract with these methods:
Method Description
sendMessage() Send a prompt and get text response with token counts
sendMessageJson() Send a prompt and get parsed JSON response
defaultModel() Returns the configured default (complex) model
fastModel() Returns the configured fast (lightweight) model
totalTokens() Calculate total tokens from a response
Provider switching is transparent to the frontend. The API endpoints, request parameters, and 
response format remain identical regardless of which AI provider is active.

Page 92. AI Job Status API
When the system operates in async mode, AI generation requests are dispatched to a 
background queue. This API allows the frontend to poll for job progress and retrieve results.
2.1 Job Lifecycle
Status Description Frontend Action
pending Job created, waiting in queue Show spinner, continue polling
processing Worker picked up the jobShow progress indicator, continue 
polling
completed Job finished successfully Stop polling, display result data
failed Job failed after max retries Stop polling, show error message
Configuration: Max 2 retries, 30-second backoff, 660-second timeout. Jobs use WithoutOverlapping 
middleware keyed by teacher_id:feature.

2.4 Polling Strategy
Recommended frontend polling implementation:
async function pollJob(jobId) {
  const maxAttempts = 60;  // 5 minutes max
  for (let i = 0; i < maxAttempts; i++) {
    const res = await fetch(`/api/ai-jobs/${jobId}`);
    const { data } = await res.json();
  
    if (data.status === "completed") return data.result;
    if (data.status === "failed") throw new Error(data.error_message);
  
    await new Promise(r => setTimeout(r, 5000));
