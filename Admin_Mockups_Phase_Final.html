08 Raport admin PUBLICATION PIPELINENEW · Status Pipeline header

Rapor moves through 4 lifecycle states (Draft → Diperiksa → Diterbitkan → Dibagikan). The new StatusPipelineStrip in the hero replaces a generic counter and lets admins jump straight into a stuck state. Per-tingkat group cards then drill into per-kelas raport readiness, with a sticky bulk-publish bar when ≥1 kelas is selected.

A · Default · Tingkat collapsed

B · Bulk publish confirm sheet

StatusPipelineStrip [NEW]

4 circular nodes connected by lines; active node enlarges to 32px and inverts to white-fill / navy-text. Tap a node → filter list to that state. Lives inside BrandPageHeader.

TingkatGroupCard

Collapsible. Header shows %-checked progress bar. Expanded view exposes per-kelas mini-chips with state color (slate / amber / red). Long-press any kelas chip → bulk select.

BulkActionBar (variant)

Reuses existing BulkActionBar with two primary slots: outline "Cetak" (PDF batch) + filled "Terbit" → opens confirm sheet B.

Confirm sheet B

Hero band itself, not a dialog. Pipeline preview row → impact list (slate cards + amber irreversible warning) → notification toggle → footer.

Empty / partial states

Tingkat with 0% checked shows red border + "Butuh perhatian" copy and a micro CTA "Lihat sub-status".

Edge: cross-semester

Period chip in hero is the single source of truth — switching period reloads the entire pipeline + tingkat list.

09 RPP admin overview REVIEW QUEUENEW · ReviewQueueColumn

Admins live in the rejection/approval loop. The new ReviewQueueColumn turns the screen into a single Kanban-like vertical with three urgency tiers (Perlu review · Sudah disetujui · Ditolak) — counts in the hero, swipeable card actions for one-tap status flip, and an inline AI-regen sheet when the rejection reason is "minta perbaiki".

A · Default · Perlu review tier expanded

B · Inline regen sheet (no full dialog)

ReviewQueueColumn [NEW]

Sectioned vertical list with tier headers (color dot + label + count). Each tier collapses to its first 5 cards with an inline "lihat semua →".

SwipeableQueueCard [NEW]

Card with a 4px left edge in tier color. Right swipe reveals approve (green) / reject (red) action. Long-press → bulk select.

QueueCountTile

Three tiles inside the hero: Perlu review (warn), Disetujui (good), Ditolak (bad). Sub-text shows weekly delta in tier-tinted accent.

RejectReasonBadge

Inline italic quote of the rejection reason on the rejected card so admins don't have to drill in to remember why.

RegenSheet (B)

Reuses AppBottomSheet + warning-tinted hero band. "Fokus regen" is a single-select chip group binding which IPK section AI should rewrite.

Edge: cooldown

If the teacher already regen'd 2× this lesson plan, the primary CTA disables and shows "Tunggu 24 jam" with a countdown chip.

10 Pengumuman admin COMPOSE + LISTNEW · AudienceMatrix

Two surfaces. The list keeps the v3 admin shell but groups posts by lifecycle: Pinned · Terjadwal · Terkirim · Draft. The compose sheet introduces the AudienceMatrix — a grid picker (rows = role, columns = tingkat/kelas) so admins can broadcast to "Wali Kelas 7+8" or "Semua guru" in two taps without reaching for nested dropdowns.

A · List · grouped by lifecycle

B · Compose · AudienceMatrix open

LifecycleChipBar

White-fill active chip + translucent inactive in hero. Pinned chip carries 📌 glyph, scheduled chip uses ⏰ in card body.

AudienceMatrix [NEW]

Rows = role (Guru / Wali Kelas / Wali Murid). Columns = Semua + tingkats + Custom. Cell tap toggles fill (navy = on). "Custom" opens nested AppFilterBottomSheet for per-kelas picking.

AudienceSummaryStrip

Live navy-tinted band at the bottom of the matrix counting computed reach: "48 guru + 8 wali kelas = 56 orang". Updates as cells flip.

PinScheduleToggle

Two stacked AdminFormToggles — neutral for "Kirim sekarang", warning amber for "📌 Sematkan" (warns in subtitle that pinning displaces other posts).

ReadReceiptBadge

"42 dari 48 dibaca · 88%" stat on terkirim cards — taps drill into a per-recipient read tracker.

Edge: empty audience

If matrix has 0 cells active, primary CTA disables and audience strip turns red: "Pilih minimal 1 audiens".

11 Kehadiran report admin SCHOOL-WIDENEW · TrendSparkRow

A pure analytics surface — no list rows, just dense aggregates. Two new patterns: AttendanceRingHero stretches the parent AttendanceRingKpi to span the entire hero, and TrendSparkRow renders a per-tingkat sparkline so admins spot dipping classes at a glance. Tapping a sparkline drills into Mockup 12.

A · Hari ini · school-wide

AttendanceRingHero [NEW]

Centered SVG ring at 64-radius inside the hero. Two-column legend brackets the ring (Hadir/Izin left, Sakit/Alpa right). Color tokens: #10B981 / #F59E0B / #3B82F6 / #DC2626.

DateRangeChipBar

Hari ini / Minggu ini / Bulan ini / Custom. Custom uses dashed outline to advertise it opens a date picker. Active chip = white fill / navy text.

TrendSparkRow [NEW]

Per-tingkat row: label · % · 7-day sparkline · delta. Sparkline color follows %: ≥90% green, 80–89% amber, <80% red. Tap row → drills to Mockup 12 filtered to that tingkat.

InlineAlertBand

Red-tinted ribbon below a row that drops >2 points week-over-week. Copy is auto-generated: "Penurunan signifikan · {worst-class} dropping fastest".

ExportSheet

The footer "Buat" opens an ActionConfirmSheet with PDF/Excel/CSV chips and a date-range echo. Email-on-export toggle uses AdminFormToggle.

Edge: weekend / holiday

If today is a non-school day, the ring fades to 40% opacity and a subtitle reads "Tidak ada sesi · libur Maulid". KPI strip pins to last school day.

12 Kehadiran detail admin DRILL DOWNNEW · CalendarHeatmap row

Drilled in from a TrendSparkRow tap. The new CalendarHeatmap shows per-student attendance at a glance — 30-day grid where each cell is a session, color-coded by status. Selecting a cell opens a correction sheet (admin-only) so HR can fix bad records without leaving the screen.

A · Per-siswa heatmap

CalendarHeatmap [NEW]

Per-student row of N cells (default 30). Cell width auto-fits row. Status colors map to #10B981 / #F59E0B / #3B82F6 / #DC2626; off-day uses #E2E8F0. Selected cell gets a 2px navy stroke.

StudentRowHeader

Avatar + name + per-student %. Card border turns red if streak ≥3 alpa. Sub-text auto-generates "⚠ 4× alpa berturut" / "↑ 1.2 vs last month".

CellDetailSheet

Inline expansion under the row when a cell is tapped (not a modal). Status chip group (mutex) + optional note + footer. Closes when the same cell is tapped again or another row is opened.

AuditTrailSidecar

Saving a correction prepends an audit row visible from the row's overflow menu — "Diubah Yahya · 14:03 · Sakit → Hadir". Read-only.

FilterChipBar

30 hari · Semester ini · Custom range · Mata pel.; tapping "Mata pel." opens an AppFilterBottomSheet filtered by what's been taught for that class.

Edge: locked period

If the cell falls outside the active semester, status chips render disabled and the helper text reads "Periode dikunci · admin SU diperlukan".

14 Sistem · Settings hub CONFIG SURFACENEW · CategoryGridHero

Settings is the only admin surface where the v3 list+filter pattern actively hurts. The new CategoryGridHero swaps it for a 2-column tile grid inside the hero, then lets each tile push to its own narrow sub-screen. Audit log is always pinned at the bottom because admins reach for it daily.

A · Hub · 6 categories

B · Tahun Ajaran sub-screen

CategoryGridHero [NEW]

2-column 170×120 tile grid below the hero. Each tile: pastel-tinted icon square + title + subline + meta. Tap → push sub-screen using same hero color but narrower height (160px).

HealthPill

"Sinkron · konfigurasi sehat" reuses the realtime indicator pattern from Mockup 02 hero. Turns amber if any setting is in invalid state.

AuditLogPin

Pinned bottom card preview of latest audit row. Always visible regardless of scroll. Tap → full audit log screen with date filters.

TimelineProgressCard

Active TA card has a progress bar with the today-marker as a circle on the bar. Auto-computes "Pekan ke-N dari M".

DangerZoneCard

Red-tinted card pinned at bottom of sub-screens that allow destructive ops (Akhiri TA, Reset semester). Outline button styling so it's never the primary affordance.

Edge: no archive

If no historic TA exists, "Riwayat" section hides entirely; the AddCTA card moves up to fill the gap.

15 Profil admin ACCOUNT + IDENTITYNEW · IdentityHero + RoleScopeChips

Two surfaces. The account sheet (peek from any header avatar) gets a denser, navy-tinted hero with role/sekolah switcher built in. The full Profil page reuses the parent ProfileScreen shell but introduces RoleScopeChips so admins can see every school they manage without leaving the page — and SecurityChecklistCard turns 2FA / fresh-password reminders into a single nudge tile.

A · Account sheet (peek)

B · Profil page · Security checklist

IdentityHero [NEW]

Sheet variant of BrandHeroSheet. Avatar + name + email + role badge cluster. Always navy gradient regardless of which role is currently active.

RoleScopeChips [NEW]

Horizontal scroll of school chips. Active school is white-fill / navy text. Adjacent chips are translucent. "+1" trailing chip opens picker if admin manages >3 schools.

RolePickerRow

For admins who also teach. Each role gets a row with icon + label + scope (e.g. "Bahasa Arab · 7B, 8B, 9C") and a "Beralih" mini-CTA. Tapping switches the entire app's role context.

SecurityChecklistCard [NEW]

Navy-bordered card with progress bar + bulleted checklist (✓ green / ! amber / × red). Items with action expose a tiny inline button (e.g. "Ganti sekarang").

ProfileFieldList

Reusable read-only data display: label (slate500 10px) + value (slate900 12px-bold). Tap row → field-edit sheet using AppEditBottomSheet.

Edge: single school admin

RoleScopeChips section hides entirely. The role picker rows still render but only show admin row.

16 Shared dialogs polish DESIGN-SYSTEMCANONICAL GALLERY

Single-page reference for every interactive admin chrome element. Use this when adding a new screen — pick from this gallery rather than inventing a new sheet/toggle/footer. Each element here is already shipped in lib/core/widgets/admin_form_components.dart; this mockup is the visual contract.

Atomic gallery

AdminFormFooter

40/60 outline-Batal + filled-Simpan. isSaving swaps primary to spinner + "Menyimpan…" and disables Batal. Samsung-safe via internal SafeArea.

AdminFormToggle

Two tones — neutral (slate text on white) and warning (amber on FFFBEB tinted card). Switch is a custom 44×24 pill with 9px white knob + 4px shadow.

AdminFormChoiceChips

Generic over T; pass List<AdminFormChoice<T>>. Tap selected chip again → calls onChanged(null) to clear.

ConfirmationDialog

Always centered icon-circle (red for delete, amber for warn, navy for confirm). Title 16/800, subtitle 11.5/500. Two-button footer matches AdminFormFooter ratios.

BulkActionBar

Navy filled bar floating above bottom nav. Up to 3 actions: 2 secondary (translucent white) + 1 primary (white fill, action-tinted text). Bottom-safe inset.

ActiveFilterChips

Lives inside BrandPageHeader. Active = white fill / navy text / value tspan. Placeholder = dashed translucent. "Reset filter" chip appears (white fill / red text) when ≥1 active.

AdminFormSheetHeader

Now supports an editingContext slot (label + initials) that renders an inline pill below the title. Replaces ad-hoc subtitle strings.

BrandPageHeader (canonical)

Hero gradient + back/actions row + breadcrumb subtitle + 22-pt title + chip strip + overlapping search bar. All admin screens start here.
